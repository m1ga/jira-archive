Description

The open & close events are not fired when touchEnabled is set to false, when the property is removed, it fires. 
var w = Ti.UI.createWindow({
	touchEnabled:false,
	backgroundColor:'red'
});
w.addEventListener('open', function( ) {
	alert('open');
});

w.addEventListener('close', function( ) {
	alert('close');
});

w.open();

setTimeout(function( ) {
	w.close();
}, 2500);

Comments

  1. Shuo Liang 2014-06-26 Hi, when you set touchEnabled: false, w will not register event any more, (include "open", "close", not just 'click' event). *All* the events will be gone. Here is a good explain may help you: http://developer.appcelerator.com/question/154004/what-is-touchenabled Regards Shuo
  2. Gertjan Smits 2014-06-26 Ok, that explains the behavior, but I assumed that only *touch* events are ignored, which I think should be the case here. An event like open or close has nothing to do with a touch to the view.
