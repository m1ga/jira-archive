  2. Chris Barber 2013-12-27 "-sdk" is not a valid option. What that is doing is setting the -s flag to true, -d flag to true, and -k flag to "C:\Program Files (x86)\Java\jdk1.7.0_25". The correct option is "--sdk" or just "-s". Furthermore, "C:\Program Files (x86)\Java\jdk1.7.0_25" is not a valid SDK. --sdk should be the Titanium SDK version to bootstrap the build, though the in the tiapp.xml always wins. If you want to set the Java home, then set it via the JAVA_HOME environment variable.
