Description

*Preconditions* User should have incorporated JMC component in his application and set it up with a valid JIRA project. *Actual Behavior* 1. User clicks on feedback icon to enter the feedback. 2. User enters feedback text. 3. User adds image to the feedback by clicking camera icon. 4. User Clicks on added image and annotates/ highlights the issues. 5. User clicks "Done" button to return back to feedback screen. 6. The keyboard should auto pop up and it doesn't. 7. Clicking on the textfield does not highlight the keyboard as well. *Expected Behavior* The keyboard should auto pop up in step 6. In step 7 after clicking text field, the keyboard should be visible for editing. *Additional Info* This issue is observed in iOS 6 and is inconsistent with previous releases. Please see when a user adds any further images, the keyboard pop back itself.
