Eric Wieber 2013-12-16 Hi Vatsal, According to Dianne Hackborn, an Android framework engineer: "They need to be set by the manufacturer (there is no way for the platform to know what they are), and as of today they are just not consistently set correctly across devices, and because nothing in the platform relies on them it is easy for devices to have them set incorrectly." The Titanium DPI calls are just calling the underlying context.getResources().getDisplayMetrics().density. These values look to be set elsewhere. You may be able to get correct values by updating your Android SDK to a higher or to the latest version. Could you please try that?
