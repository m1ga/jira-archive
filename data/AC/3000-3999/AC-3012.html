Description

Normal iOS behaviour is for a 'slide' animation to occur when a new window is opened via a NavigationGroup and when the iOS native 'back' button is clicked to close the window. It seems that something is broken when running in iOS5 and/or built under recent versions of Titanium builder. This is also apparent in Kitchensink's Nav Group demos when run under iOS5 on my iPod. The animation works for window open and doesn't when closing it. The window simply closes in the blink of an eye.
