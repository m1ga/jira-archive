Description

Steps to Reproduce

Appcelerator running on Windows 7 (64-bit) When you set the 'Titanium SDK Home' = 'C:\Users\\AppData\Roaming\Titanium' The Appcelerator complains that '.....AppData\Roaming\Titanium\sdk\win32' is not a directory. Under the set directory, there are only mobilesdk & modules sub-directories, but not the sdk the error is referring to. When you change mobilesdk to sdk and restart the application. No error is reported in the Window->Preferences setting. But the system detect missing mobilesdk and proceed to recreate the missing mobilesdk directory. Its seems that the path in error checking and sdk directory update are not in sync. http://stackoverflow.com/questions/10556575/how-to-locate-titanium-mobile-sdk-in-titanium-studio http://developer.appcelerator.com/question/129786/could-not-locate-a-titanium-desktop-sdk-at-the-given-path

Actual Result

see above.

Expected Result

Both the error checking path and sdk update should be referring to the same path?
