Description

openURL("tel:xxxxxxxxxx") command does not send paused tel: dial string from Android For iPhone the call would be "tel:07944444444,1111" - which works However for Anroid when trying "tel:07944444444,1111" and "tel:07944444444p1111" neither works "tel:07944444444,1111" just sends the first part - ie. 07966794444 but does not include the section after the comma "tel:07944444444p1111" removes the 'p' and sends the whole string as one - ie. 079444444441111 This functionality is required for functions such as calling card applications, conference call dialling, and automated extension calling Is there a way to get this bug resolved/functionality built in?
