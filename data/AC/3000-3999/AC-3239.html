Description

When configuring data binding for a TableView, if binding the "title" property on a TableViewRow to a model property, and the model's property is not a string, an argument exception is thrown and the application crashes. I would think that the proxy should accept values not of string type and just convert them. [ERROR] The application has crashed with an uncaught exception 'NSInvalidArgumentException'.
