Description

Currently ID fields default to using randomly generated GUIDs stored as TEXT fields. It would be better to use SQLite INT PRIMARY KEY AUTOINCREMENT fields, which map to the underlying SQLite ROWID, at this would have the following improvements over the existing GUID implementation: 1. no chance of random id value collision (admittedly it's pretty small currently, but not non-existant). 2. require considerably less storage space (both in field size and in index size) 3. more performant on lookups.
