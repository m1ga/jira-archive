  1. Chris Barber 2016-08-17 I'm not sure if this is really a bug or not. When we detect a crash, we wait a whopping 1000ms for the crash log to show up. If it doesn't exist after waiting a second, then it assumes there's no crash log and hence the error. Probably a better way is to check for a crash file every 250ms for up to 5 seconds (so checks 20 times) to see if it exists. If it doesn't show up, then we will continue to say "Detected crash, but no crash file."
