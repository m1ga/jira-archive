Hans Kn√∂chel 2017-11-10 It's a behavior used for username / password fields and seems to be a known problem for devs, see [here](https://stackoverflow.com/questions/45452170/ios-11-disable-password-autofill-accessory-view-option). Not sure what's the best thing to address this, but it doesn't really look like a Titanium issue. Applying the workaround provided in Stackoverflow (again, this is not specific to Titanium), you can set the "autofillType" property of the text-field to an empty string, e.g 
       field.autofillType = '';
