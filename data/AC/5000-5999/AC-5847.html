  6. Chris Barber 2018-08-20 [~sdarda] That's great that you got it working. The problem you ran into, as indicated in your log output, is a permission issue. I recommend that you never run "npm install" with "sudo" unless you are specifying the -g global flag. The problem with using "sudo" when installing something in your user directory is the files will be owned by root instead of your user. This is especially bad when installing npm packages since the packages get cached and are owned by root. You should probably fix the file permissions in your npm cache.
