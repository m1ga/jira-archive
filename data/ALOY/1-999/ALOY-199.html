Description

To build the Alloy docs using JSDuck, we need to comment the classes that should be exposed, and remove any comments that look like doc comments--that is if we do this: 
/**
 * Do something here.
 */
 exports._ = function() ...
JSDuck will add a global method called "_" with the description "Do something here." Changing the doc comment intro /\*\* into a regular comment intro /\* eliminates the rogue method. Currently, it looks like we should be building docs from Alloy/builtins and Alloy/lib.
