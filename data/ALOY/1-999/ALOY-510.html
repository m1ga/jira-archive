Description

Problem description

Not possible to specify the iOS certificates and run the app con device when running 'alloy run -p ios -T device'

Steps

From Terminal in any project folder, run 'alloy run -p ios -T device'. The user is informed that the command is actually deprecated and to use 'titanium build' instead (alloy run will be removed with 1.1.0). The user is then prompted with which 'iOS Developer Certificate' to use. However, the cursor is not placed in the same line and pressing the return key does not generate the list of the available certificates, nor fillin in the certificate name produce any consequence. Using 'ti build -p ios -T device' works fine as expected.
