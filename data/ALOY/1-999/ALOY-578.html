Description

A had a string literal that contains escaped quotes: var JSONtest = JSON.parse('{"result": "{\"Xzibit\":{\"JSON\":\"within JSON\"}}", "id": "7ca16380-56c5-49d4-bfb3-55d8c8f9177e" }'); When the parser parsed it, it replaced the single-quotes with double quotes, and escape the qouble-quotes that weren't already escaped, but didn't escape the escaping of the already escaped quotes. The resources version of the code was this: var JSONtest = JSON.parse("{\"result\": \"{\"Xzibit\":{\"JSON\":\"within JSON\"}}\", \"id\": \"7ca16380-56c5-49d4-bfb3-55d8c8f9177e\" }"); when it should have been this (note some places have three backslashes): var JSONtest = JSON.parse("{\"result\": \"{\\\"Xzibit\\\":{\\\"JSON\":\\\"within JSON\\\"}}\", \"id\": \"7ca16380-56c5-49d4-bfb3-55d8c8f9177e\" }"); Marking as minor as (a) There is a workaround, and (b) this code only written to test another issue with my code.
