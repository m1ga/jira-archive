Description

Right now styles are only available at compile time. We need to make the styles available at runtime so that they can be manipulated due to user interactions or other runtime information. This will involve converting some of the most complex code in alloy's node.js code into Titanium compatible code. Fortunately most of it should be pure JS, but undoubtedly this will take some time. The generated, ordered style arrays created during the alloy compile phase should be made available in runtime modules. The modules should simply make these ordered arrays available at runtime and a separate API will be used to operate on them in order to do runtime styling of UI components. The style modules should correspond in name to their runtime controllers and will be housed in the "Resources/alloy/styles" folder. For example, the runtime style module for the index controller woud be found at "Resources/alloy/styles/index.js"
