  2. Fokke Zandbergen 2015-01-09 The correct way to escape (double) quotes in XML according to the [specification](http://www.w3.org/TR/REC-xml/#syntax) is to use " or use in a text node. I think with this argument this ticket can be closed as *Won't Fix*
  3. Tim Poulsen 2015-01-09 Resolving per Fokke's comments. Escaped characters aren't supported in XML, we don't support CDATA blocks, and there's no reliable way to replace quotes (escaped or otherwise) in attribute values given the many possible permutations of strings developers might use. Additional workarounds would be to put the strings into the TSS or JS, or for quotes within HTML (not surrounding attributes as in the original problem description) would be to use entities like &ldquo;
