  1. Ivan Skugor 2012-07-17 Note that this is not iOS only issue. Android also has issue with "postlayout" event, but it is different (maybe another ticket should be filed for that).
  2. Ivan Skugor 2012-07-18 Sorry, I missed to notice that other "Android" ticket. It is incomplete, as you can see in this ticket, "postlayout" event is not fired for TiButton and TiLabel.
  3. Stephen Tramer 2012-07-24 We cannot fire only a single "postlayout" event in all situations because the multiple layout passes are required and we cannot determine which one is the final event without looking into the future. Instead the number of postlayout events can be reduced, so they are only fired when the view's size actually changes.
  4. Ivan Skugor 2012-07-25 It would be nice if situations when events are fired multiple times are documented. Thanks.
  5. Vishal Duggal 2012-07-25 PR https://github.com/appcelerator/titanium_mobile/pull/2632 merged against master. As noted in the comments the postLayout event is only fired if either the size or position of the view changes. Also the event no longer propagates to the parent.
