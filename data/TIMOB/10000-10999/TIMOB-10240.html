Description

It would be nice if the deprecation annotation included a free-text field where we could describe what to use instead. The alternative is to reuse the summary text, possibly removing the only documentation describing the item. This proposal is to add a field, which for the sake of argument I'll call notes, like this: 
deprecated:
    since: 1.8.0
    removed: 1.9.0
    notes: Use  instead.
For summary tables, the deprecated.notes field could be used in place of the summary, so users would see something like: bq. DEPRECATED Use instead. For the detail page, the note would show up as part of the deprecation warning, and the usual summary would appear: bq. Deprecated since 1.8.0, removed in 1.9.0. Use instead. bq. bq. Summary bq. A view which automatically renders as an octagon.
