Description

In the mobileweb, photos in the Chat disappear when a chat message is sent. On Android and iOS devices, the photos are visible even when a new Chat message is sent. This is not regression, this occurs in 2.1.2.GA also. Note: The ti.cloud app is not designed to upload photos from mobile web. This issue is only about existing pictures in the Chat that disappear once a chat is sent using mobile web. Steps to reproduce: 1. In the Studio, clean the project and compile it using "Mobile Web Preview in Browser" 2. Go to Users > Login User 3. Enter "smappcel@gmail.com" in "Login" field 4. Enter "1234" in the "Password" field, click "Login User" button and go back to the root level 5. Now click on "Chats", click on the chat group(sm app, sm g) 6. Observe the chat view which has photos in the chat 7. Now enter a chat message Actual result: When a chat is entered using mobile web, all the pictures in the chat disappear. This only happens on mobile web. On iOS and Android devices, the photos are still visible after sending the chat. Please see the attached screenshots. Expected result: When a chat is sent using mobile web, the pictures in the chat should still be visible. Please use the attached cloud project "C1236".
