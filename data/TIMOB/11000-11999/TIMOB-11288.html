Description

This is not a regression. It occurs since 2.0.1 Running the app 'Menu handlers(Activity Property)' second time force closes the kitchen sink. Crash Logs are attached. Steps to reproduce: 1) Install the refactored kitchen sink on android device and run it. 2) Go to Base UI -> Menu(Android) -> Menu handlers(Activity Property) 3) Now press back button and then again go to Base UI -> Menu(Android) -> Menu handlers(Activity Property) Expected result: 1) The app should run successfully over device. 2) It should display window with a button 'Open Window'. 3) It should again display the same window as in step 2. Actual result: 1) The app runs successfully over device. 2) It displays window with a button 'Open Window'. 3) It force closes the app. Logs attached with bug.
