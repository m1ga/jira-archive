Chris Barber 2013-07-25 @Russell, I just tested building an app with the bleeding edge Titanium CLI, Node.js 0.10.12, a selected SDK of 3.1.1.GA, and without a in the tiapp.xml and it built correctly using 3.1.1.GA. The latest stable CLI should work too. The only tiapp.xml fields that are required are name, id, and guid. The error described above is because they were using an old Titanium SDK (3.0.0 or 3.0.2) and Node.js 0.10.x. This means that if you want to build using Titanium SDK 3.0.0 or 3.0.2, you have to have Node.js 0.8.x installed. If you want to build for 3.1.0 or newer, you can use Node.js 0.8.x or 0.10.x.
