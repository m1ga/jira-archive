Description

If the user wishes to exit an app that is being debugged, the back button will not terminate the session, the app is locked until terminated from studio. The user should be able to close the app without terminating the session from studio. This occurs on both emulator and device. This does not occur in iOS. This is a parity issue. This is a usability issue. Steps to Reproduce: 1. Launch app with a breakpoint in debug mode on Android. 2. After the breakpoint is hit, press the back button. Actual Result: Nothing happens, the app is locked. Expected Result: The app terminates the debug session and closes.
