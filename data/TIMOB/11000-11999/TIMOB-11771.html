Description

The problem

WebView component crashes Android emulator even when basic HTML is loaded. This makes testing of app with WebView imposible. :(

Test case

var win = Ti.UI.createWindow();

var webView = Ti.UI.createWebView();

webView.html = '<html><head><title></title></head><body>Testing testing .... </body></html>';

win.add(webView);

win.open();

  3. Nathan Nadeau 2012-11-17 I have also run across this exact problem of the last few days so I wanted to add some more information. The problem also happens if you set url property instead of html property of WebView. Of course there are ways to increase the JNI limit beyond 2000, but that does not really solve the leak.
