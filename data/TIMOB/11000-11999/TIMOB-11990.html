Description

While testing packaging with icons and artwork, iTunesArtwork PNG when placed in the project /resources/iphone folder works, but when placing it in the root of project does not get copied into the ipa correctly. This does not occur with 2.1.4 GA Steps to reproduce: 1) Create a project with a 512x512 image in png format called "iTunesArtwork" at the root of the project with the ".png" removed from the name via either the finder or studio rename. 2) Package for adhoc or distribution 2.1) if Distribution, re-sign from the archive with an ad-hoc provision 3) Drop the file into iTunes and check the icon Result: The icon is missing Expected: The icon is present
