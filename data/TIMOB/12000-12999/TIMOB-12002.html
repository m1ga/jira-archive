Description

Since "iphone", "ipad", and "ios" all do the same thing, it's best to remove "iphone" and "ipad" to remove confusion. Use the device targets in the tiapp.xml or --device-family option to specify "iphone", "ipad", or "universal".

  5. Chris Barber 2013-01-16 This is expected behavior. Nowhere does the CLI advertise that "iphone" or "ipad" are valid build targets, but for a period of time we did allow them. This ticket exists because it doesn't make sense to have "iphone", "ipad", and "ios" targets that are all identical. Too confusing. The reason we still allow "iphone" and "ipad" is because there are versions of Titanium Studio that may still be in use that pass in "iphone" or "ipad" instead of "ios". Someday we'll rip out support for "iphone" and "ipad", but it doesn't hurt anything to leave it in.
