Description

When doing a iOS dist-adhoc or dist-appstore build, you need to specify a --distribution-name. You can save your distribution name in the CLI config file so that you don't need to specify it every time. However, there was a bug that was introduced where the distribution name from the CLI config is only applied if the name was save in all lower case. 
titanium config ios.distributionName "Appcelerator, Inc." # won't work

titanium config ios.distributionName "appcelerator, inc." # works!
This only is a problem if you DO NOT specify the --distribution-name at the command line and the default is being applied. This is not an issue for Studio since they always pass in the distribution name.
