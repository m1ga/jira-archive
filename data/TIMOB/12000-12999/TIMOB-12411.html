Description

Problem description

Calling Ti.Android.getCurrentActivity() always fails with an exception in the logs. 
E/TiJSError( 1288): (main) [2,1195] - Message: Uncaught TypeError: Object # has no method 'getCurrentActivity'

Steps to reproduce

Use this simple app to test: 
Titanium.UI.setBackgroundColor('#000');

var win1 = Titanium.UI.createWindow({
    backgroundColor:'#fff',
    fullscreen: true
});
win1.open();

Titanium.API.info("Activity: " + Titanium.Android.getCurrentActivity());
Changing from getCurrentActivity() to currentActivity (direct access to the property) works fine.

Gary Mathews 2015-02-11 Both _Titanium.Android.getCurrentActivity()_ and _Titanium.Android.getCurrentService_ have been removed from the documentation. _Titanium.Android.currentActivity_ and _Titanium.Android.currentService_ should be used instead. This bug is because docgen automatically generates getter methods for properties even if the getter is not implemented.
