Comments

  1. Praveen Innamuri 2013-03-26 When Tizen app is created from Studio, it creates a default template URL "http://" in the URL field of TiApp. Tizen CLI fails to parse this url while installing the app on device and thus, the launch fails. Note that this field is populated for all apps. If the TiApp is modified to remove the URL, then the app launches on device without any problems.
  2. Anatoliy Odukha 2013-03-26 Does it work after you changed "http://" to valid url? Could you check device firmware version and attach wgt please? I repeated steps. There was "http://" in url. But I failed to reproduce it. Applications runs well on tizen device.
