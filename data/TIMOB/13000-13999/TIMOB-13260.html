Description

Based on the page here: https://wiki.appcelerator.org/display/mds/Module+Planning+Page There is a suite of modules that we wish to ship with the SDK by default. This story encapsulates the idea of setting up the build, test and verification infrastructure to make such a change happen. Currently, modules shipped with the SDK are packaged as .zip files and checked in. We need amore flexible approach, likely one that packages a known good version of the module at build time.
