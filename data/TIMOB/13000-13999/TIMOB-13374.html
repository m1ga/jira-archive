Description

Description

We noticed an issue in studio with color codes in the console. After further investigation, this turns out to be a regression from 3.0.24 in the CLI. The --no-colors flag is not honored, so all activity is output in color. This presents a problem more so in Studio where the text includes codes for the colors and therefore looks "non-ideal". This is a CLI regression from 3.0.24

Steps to reproduce:

1) Try any CLI command with the --no-color flag (I like to use "ti config --no-colors")

Result:

Colors still present in the output

Expected result:

No colors in the output
