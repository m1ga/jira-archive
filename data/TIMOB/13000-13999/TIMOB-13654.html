Description

Problem description

When switching between different projects or different platform/emulators/devices, LiveView server might stop working as expected. In order to have it working consistently the following command to first stop the server has to be run 
liveview server stop

Comments

  1. Michael Xia 2013-08-14 Verified on Appcelerator Studio, build: 3.1.2.201308091728 (containing LiveView 0.1.32), OSX 10.8.3 that launching different projects with LiveView on iOS simulator worked successfully without a need for explicitly stopping the server. Closing.
