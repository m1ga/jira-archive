Description

When you download a module from the marketplace or github, in order to have it work in a project, you need to add it to that project. But "Core" Modules, which really are part of our SDK, you should not have to "add" the module to the project. If you use the API, then the module should automatically be added to the project. If the the API is not used, then it shouldn't be there. "Core" module should work like regular APIs, or as close as possible to regular APIs. The end user shouldn't have to know that they are built as modules. Today they are provided on the marketplace, but in the near future they will be packaged with the SDK.
