Description

Description:

When testing liveview, it is apparent that launching a liveview session then switching projects will continue to show the first project used. The workaround for this is to manually kill the server then launch the new project. With our delivery mechanism being through studio, this is not an easy command to find, located deep in the Appcelerator studio plugin folder.

Steps to reproduce:

1) Launch a liveview simulator session with a project 2) Close the simulator or terminate this and other processes through studio 3) Switch to another project and launch that with liveview

Result:

The new project is not shown

Expected Result:

The project you launched will be shown
