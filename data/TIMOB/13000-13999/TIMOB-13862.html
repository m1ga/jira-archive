Description

If the tiapp.xml is set to 3.1.1.v20130514180723, but the CLI is set to 3.1.1.v20130513120107, since both start with 3.1.1, the CLI thinks they are the same version which is wrong.

Chris Barber 2013-08-27 To test: 1) Install 2 different CI builds (I tested with 3.1.1, but the 3.1.3 builds should work fine... you just need to be sure the X.Y.Z are the same and the timestamps are different) 2) Select one of the CI builds as the selected sdk 3) Assign the other CI build as your app's in the tiapp.xml 4) Build and notice it should properly fork the correct CI build (instead of using the selected one)
