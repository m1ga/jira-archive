  1. Russell McMahon 2013-05-29 For BlackBerry simulator you can not use a loopback address (127.0.0.1) to test servers. Best to use a separate machine. 1) Run the testserver.js on a different machine then running the simulator. 2) Do ifconfig from command line on machine running the server and note the ip address of the main interface receiving internet connections. 3) From machine running BB simulator ping simulator address to make sure available. 4) Run includes js test sample but make sure the address is the ip address of machine running server and not 127.0.0.1
  2. Lokesh Choudhary 2013-05-29 Verified the fix with the steps mentioned in the comments by russ & I can see the custom header in the logs. Thus closing.
