Description

Steps to reproduce : 1) Configure Tizen SDK with invalid keystore/certificate or with an invalid password. 2) Run Tizen app in Emulator. The console log indicates that the install command failed and thus, the tizen app failed to launch in emulator. There is no indication about the invalid keystore/password configuration that has lead to the failure of the app launch. In case if the certificates are expired or if the user has accidentally entered the wrong password, then there is no way to figure out the launch failure is due to the configuration.
