Description

Application specific Console logs does not appear for iOS Simulator 7.0: Retina & Tall (4-inch 64-bit). Console logs appear for iOS simulator 7.0 Retina. This is not regression issue. This issue is specific to 64bit simulator. Steps to reproduce: 1. Install kitchensink on iOS 7.0 device or Simulator v7.0 2. Go to Kitchensink. Right click >Run as >Run Config > 64 bit iphone simulator. Change simulator config to "Retina & Tall (4-inch 64-bit)". OR iOS Simulator>Hardware>Device>iPhone Retina(4-inch 64-bit) 3. Go to KS>Controls>Progress Bar 4. Click "Start Progress" button Actual: Console does not show following info: [INFO] INTERVAL FIRED value 0 [INFO] INTERVAL FIRED value 1 [INFO] INTERVAL FIRED value 2 Logs do not appear in 64bit for: Ti.API.info('INTERVAL FIRED value ' + val); Ti.API.debug('INTERVAL FIRED value ' + val); Ti.API.warn('INTERVAL FIRED value ' + val); Ti.API.error('INTERVAL FIRED value ' + val); Ti.API.trace('INTERVAL FIRED value ' + val); Expected: The above info must appear in console. Changing iphone simulator to "Retina" will display above info
