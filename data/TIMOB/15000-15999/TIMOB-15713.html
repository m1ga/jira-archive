Description

When opening the Xcode project after an iOS build (any target), you are unable to build the project due to CLI errors with trying to prompt for the missing options (which in itself is also a bug since it should not be prompting in the first place). Steps to reproduce: 1) create a project in Studio or from the CLI 2) build for ios sim in Studio or from the CLI 3) exit sim/build/studio 4) open xcode project in build/iphone dir 5) build project in xcode for ios sim You can also try the same thing for device builds. Actual: Titanium CLI crash in the Xcode build log during pre-compile phase. Expected: Xcode project to build without errors or dragons.
