Description

When a 3.1.x SDK is selected, the legacy Xcode info prints that it's not supported and the Titanium SDK version is undefined. The problem is this text should not have been rendered at all. Expected: 
Xcode
5.0.2 (build 5A3005) - Xcode default
  Install Location                      = /Applications/Xcode.app/Contents/Developer
  iOS SDKs                              = 7.0
  iOS Simulators                        = 6.1, 7.0
4.6.3 (build 4H1503)
  Install Location                      = /Applications/Xcode46.app/Contents/Developer
  iOS SDKs                              = 6.1
  iOS Simulators                        = 5.0, 5.1, 6.0, 6.1
Actual: 
Xcode
5.0.2 (build 5A3005) - Xcode default
  Install Location                      = /Applications/Xcode.app/Contents/Developer
  iOS SDKs                              = 7.0
  iOS Simulators                        = 6.1, 7.0
  Supported by TiSDK undefined          = no
4.6.3 (build 4H1503)
  Install Location                      = /Applications/Xcode46.app/Contents/Developer
  iOS SDKs                              = 6.1
  iOS Simulators                        = 5.0, 5.1, 6.0, 6.1
  Supported by TiSDK undefined          = no
To reproduce: 1) Select a 3.1.x SDK: "ti sdk select 3.1.4.GA" 2) Run "ti info -t ios" 3) Look at the Xcode info section
