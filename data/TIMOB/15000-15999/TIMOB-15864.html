Description

When trying to build a module with 'ant', it fails when the Android NDK path contains spaces. The module build scripts aren't directly related to the CLI, but the CLI should validate the NDK path during the setup command to help avoid this scenario. This should be fixed in both "ti setup" and the Android detection code in the SDK.

Comments

  1. Chris Barber 2016-09-02 Using ant for building Android modules has been deprecated in favor of the new module build added in Titanium SDK 4.0.0.
