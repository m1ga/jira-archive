  5. Andrew Tokmakoff 2012-11-05 It seems that the simulator has trouble with the URL given to it having spaces and the URL needs to be escaped. If you run the project directly from XCode, it defaults to using a different "temp url" for the recorded audio and that one has no spaces. For me, that is: /var/folders/3q/vv7snyc13259jyb_795x2wmw0000gn/T as opposed to something like: "/Users/onlee/Library/Application Support/iPhone Simulator/" I followed the fix outlined here: http://lists.apple.com/archives/coreaudio-api/2010/Aug/msg00210.html and it works now in the simulator and the -50 error code is gone.
