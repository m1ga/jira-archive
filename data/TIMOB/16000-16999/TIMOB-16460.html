Description

Description

Creating a default application for Windows Hybrid works successfully through CLI. Building to device and emulator works fine too. Packaging the application for distribution also works fine. However after running the Windows App Certification Kit, errors start to show. I have attached the results of the test to this ticket.

Steps To Reproduce

0. Make sure you have Windows App Certification Kit installed. 1. Create a default windows hybrid application 2. Build the application for *winstore* 3. Open the generated solution file in the build directory 4. Once open in Visual Studio, click Project -> Studio -> Create App Packages. 5. Run the Windows App Certification Kit on the packaged build.

Expected Result

Should successfully pass the tests.

Actual Result

Errors are detected in the code. +Extra Information+ Windows App Certification Kit runs a series of tests against the users application to make sure it is suitable for the windows store, putting it through multiple stress tests.
