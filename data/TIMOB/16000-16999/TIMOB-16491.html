Description

Icons are not being copied to the correct location when a project is built. Icons are being copied to "Apps/" when the should be in "images/" I have attached a project with a few crude images and their expected file names and sizes. Windows will ignore any images you have saved in the "Apps/" folder and displays the windows default app-icon, and the only way to correct the icons is to manually copy the icons into the correct location and then build and deploy the project using the generated visual studio solution file. +Extra Information+ The following link is the official windows documentation for file names and icon sizes: http://msdn.microsoft.com/en-us/library/windows/apps/hh846296.aspx
