Description

We should revive/dust off ti.worker for ti.current async functionality. Adding first-class async support to ti.current would be an involved undertaking for iOS (but easier for Android). We will "do it right" in ti.next. This is probably 2-3 days effort. Breaking (and unnecessarily complicated) changes were introduced to the SDK that broke the ti.worker module. We need to undo these SDK changes, which is basically code cleanup.
