  4. Hieu Pham 2014-06-05 This isn't a bug. Blur is fired but since the TextArea is the only focusable widget on screen, it gets refocus by Android. Since the label updates whenever blur or focus fires, focus text overrides blur text, creating the illusion that blur never fire. Basically when you click on 'blur', these events will fire: blur -> focus. I suggest change the test case to log whenever these events fire, instead of using a label to display the latest event that is fired.
