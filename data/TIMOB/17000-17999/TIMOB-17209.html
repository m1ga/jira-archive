  1. Chris Barber 2014-06-25 Titanium 3.3.0 has new logic for creating module projects and are now subject to the same rules as creating apps. The error message is by design and is correct.
  2. Satyam Sekhri 2014-06-25 As the restriction for not allowing number after a period in Module ID of a module is driven by the restriction on Android platform, the restriction can be made same as in case of creating apps. During app creation, the app id cannot have a number after a period only if one of the target platforms is Android. It fails with the same error message as the one mentioned above for module creation. However, if Android is not in the list of target platforms then only a warning is shown for such app id and app can be created successfully.
  3. Paras Mishra 2014-07-17 Closing the issue as result is expected behavior.
