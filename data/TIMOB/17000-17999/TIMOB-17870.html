NOTE: This is not an appcelerator sdk issue. This is due to some other potential compatibility issue with HAXM & Yosemite.

I have opened this ticket so that we have it in jira.

Description:

1. We are seeing an issue where HAXM is not working on Yosemite GM candidate 2. 2. Even though HAXM is installed & the processor supports virtualization technologies the x86 emulators for android fails to launch. 3. We get the following error in the android avd manager when we try to launch an x86 emulator: 
Starting emulator for AVD '5.0_X86_32bit'
emulator: ERROR: x86 emulation currently requires hardware acceleration!
Please ensure Intel HAXM is properly installed and usable.
CPU acceleration status: HAX is not installed on this machine (/dev/HAX is missing).
4. This issue is not seen on Maverics 10.9.5 which is dual booted along with Yosemite GM candidate 2 on the same machine.

Comments

  1. Lokesh Choudhary 2014-11-12 This issue has been fixed by the HAXM update to ver.1.1.1 which comes with android tools. Now, we can successfully launch x86 32bit & x86 64bit emulators on Yosemite 10.10. Closing.
