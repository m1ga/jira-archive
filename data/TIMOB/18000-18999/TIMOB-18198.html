Description

*Details:* NavWin will stutter if you: select a TableViewRow that opens a window, which contains a ScrollView, which contains several TextFields, and focuses on the first TextField. *Note:* This is *not a regression* as this occurs in 3.4.1.GA. *Steps to reproduce:* 1. Create a Titanium classic project with the app.js attachment 2. Install app to device 3. Press on the first row *Actual:* The NavWin will stutter; see recording.mov. In the recording, after the row is selected, the window will go blank for a couple of seconds. This part *does not occur* on device. *Expected:* The NavWin should not stutter.
