Description

There's two issues with the generated CMakeLists.txt from our template: - nested folders are using slashes in the variable name, rather than replacing all slashes with underscores (only the first is getting replaced, not all slashes) - The incorrect slash is being used to designate the destination relative path inside the app. We need to use '\', but are using '/'. As a result when we try to build the app in Visual Studio it will complain about being unable to find files that do exist.
