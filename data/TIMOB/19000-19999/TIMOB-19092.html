Description

Description

When building a project that already has a generated pfx file then a new pfx can't be generated until the files are deleted from the folder. As per the prompt Where is the pfx file used to sign the app? (leave blank to generate) \[generated.pfx]: by leaving this blank a new pfx should be generated but the default is used instead.

Steps to reproduce

1. Build a project that has an already generated pfx file 2. When prompted for the file, leave the prompt blank

Actual result

The existing file will be used

Expected result

As per the prompt a new file should be created
