Description

When running an app on a simulator, terminal requests permission to control the computer. The simulator also performs a 'Hardware -> External Display -> Disabled' action. Is this required? Can these be removed? *Steps to reproduce issue* 1. Create a new project from the CLI appc new 2. Run the project on an iOS simulator appc run -p ios *Actual Results* Terminal requests access and the simulator performs 'Hardware -> External Display -> Disabled' *Expected Results* The app can run on simulator without requesting access or performing menu actions. *Notes* This appears to only occur with simulators that do not support Watchkit
