Ewan Harris 2015-09-11 Reopening ticket: Windows 8.1 Appc CLI Core: 5.0.0-50 Appc CLI NPM: 4.2.0-1 Titanium SDK : 5.0.0.v20150911123911 Lumia 930 (8.1) When using the below code I do not see events get fired for for *touchstart* and *touchend* and the behaviour for *longpress* differs to Android and iOS. On Windows when listening for the *longpress* event the callback will be called twice, when the duration for a longpress is met and when the press is released, on iOS and Android a longpress is only fired once, when the length of time is met. 
       var win = Ti.UI.createWindow({ backgroundColor: 'blue' });
   
   var lpview = Ti.UI.createView({
       backgroundColor: 'red',
       width: 100, height:100,
       top: 10
   });
   
   
   lpview.addEventListener('longpress', function(e){
   	console.log('You longpressed me!');
   	console.log(JSON.stringify(e));
   });
   
   var touchview = Ti.UI.createView({
       backgroundColor: 'purple',
       width: 100, height:100,
       top: 150
   });
   
   
   touchview.addEventListener('touchstart', function(e){
   	console.log('You touchstart me!');
   	console.log(JSON.stringify(e));
   });
   
   touchview.addEventListener('touchend', function(e){
   	console.log('You touchend me!');
   	console.log(JSON.stringify(e));
   });
   
   win.add(lpview);
   win.add(touchview);
   
   win.open()
