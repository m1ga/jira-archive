Description

If you're using native API access and create something like a Windows.UI.Xaml.Controls.Button, you'd naturally want to set a text label for the button. But the Content property is a Platform.Object with the understanding it could be a String or it could be an UIElement (in fact it could be a UIElement, and if it's not then it's basically converted to a String representation, though how that's done isn't really talked about). We need to understand this nuance of the API and support settings string values easily, i.e. 
button.Content = "Click me!";
https://msdn.microsoft.com/en-us/library/windows/apps/windows.ui.xaml.controls.contentcontrol.content
