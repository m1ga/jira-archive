Description

This seems new as of 4.2.1 and the noted build of TiMobile (1.5.0.3ee6a97). During the pass I encountered a number of bugs that I believe are related to this.

Hard to repro, but when it occurs, this is what it looks like:
1.tapping on list view will shows the highlight, but will not seem to load the targetted js file (however, traces show the code is being executed)
2.tap a second time
results: test loads
3.tap back button
results: one view is replced by a second
4.tap back again

results: finally back to tableView list

Only seen on 4.2.1 so far.
