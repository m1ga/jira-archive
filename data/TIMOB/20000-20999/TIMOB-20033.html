Description

Currently when you do a third party module that required a .bundle file, you need to tell the users to copy the file manually to /platform/iphone, because the cli does not detect the file in /modules/iphone/{{my_module/platform/iphone}}. Would really be great to support this! The workaround is to add a hook inside a plugin, but the user experience would be improved by doing it manually.

Comments

  1. Chris Barber 2015-11-24 Resolving as invalid. The Titanium iOS build does copy the <project-dir>/modules/iphone/my_module/platform/iphone files into the <project-dir>/build/iphone/build/Products/Debug-iphonesimulator/appname.app directory, and perhaps this is why you think they aren't being seen. I've changed this behavior so that files in <project-dir>/platform/ios and <project-dir>/modules/iphone/my_module/platform/iphone are copied into the <project-dir>/build/iphone directory as apart of TIMOB-19694.
