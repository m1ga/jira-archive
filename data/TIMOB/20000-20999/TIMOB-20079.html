Description

As described here: https://docs.appcelerator.com/platform/latest/#!/api/Titanium.UI.iPad.Popover Ti.UI.iPad.Popover doesn't have backgroundColor property. Current iOS native implementation for the iPad popover allows the change of background color as described in this documentation: https://developer.apple.com/library/ios/documentation/UIKit/Reference/UIPopoverPresentationController_class/#//apple_ref/occ/instp/UIPopoverPresentationController/backgroundColor

Comments

  1. Rick Blalock 2015-11-30 You can change the background color of the popover as long as you're passing in a window with a background color. However, the tail of the popover does not match, here's a video: https://db.tt/gNADVBg8
  2. Hans Knöchel 2015-11-30 Allowing to set the backgroundColor inside the popover directly instead of its content view to ensure the coloring of the "tick" as well. We document the property as creation-only. PR: https://github.com/appcelerator/titanium_mobile/pull/7513
  3. Chee Kiat Ng 2015-12-01 CR and FT Passed. PR merged.
  4. Harry Bryant 2016-01-19 Verified as fixed, setting the background color directly within the popover now correctly sets the popover background & the "tail" of the popover.
