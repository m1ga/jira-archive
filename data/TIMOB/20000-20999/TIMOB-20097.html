Description

When trying to deploy a Win 10 mobile app, they now require certs/keys like Windows Store apps do. We need to detect when there's no pfx and generate one like we do for store (or cheat and copy the temporary one from cmake's Templates/Windows folder for them to use). We should probably just use the temporary key for dev, and have them generate an actual pfx for publishing/deploying.
