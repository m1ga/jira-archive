  3. Angel Petkov 2016-01-21 After investigating the issue , it seems the problem is native. In our SDK all we do is synthesize the arguments and pass them to the delegate. It seems to wok with all other .doc files that are not exported from word 97. After doing some more digging it seems that Apple have the same issue with more of their software when it comes to reading .doc file that are Microsoft 97-2004. It appears to be an issue with Email attachments and Page app. A solution would be to simply put all the context of the so doc file and move it to a newer version of MS Word. I would have suggested to also add the file to an online doc [viewer](http://www.viewdocsonline.com) and then add the link to our webView. However it seems not even onlineDocs will read the file format.
