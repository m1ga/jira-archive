Description

When I build an app without setting a minimum iOS version it defaults to 7.1. In that case, I noticed launch images for iPhone 6+ were missing. This is caused by the following commit: https://github.com/appcelerator/titanium_mobile/commit/105a1fdf72e8f349f71ba10c08c738d9160910ec#diff-f430483233aa01af5b10df8390f9635dR4931 Which is related to the storyboard ticket TIMOB-19694 I don't get the logic for this validation of minIosVer. If that is the minimum iOS version the launch image is supported on, then surely it should be removed when you'd set a *max* iOS version (which I don't even think you can't). If you build to support iOS 7+ then of course the launch images for 8+ *should* be included, right?
