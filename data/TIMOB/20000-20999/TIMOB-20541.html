  10. Samie 2016-03-10 Hey everyone, Some good news, I have managed to solve this problem! Basically this problem occurs if your XCODE version is not the latest version (currently version 7.2.1)! Here is the catch, to install that version if you are on a mac you will need to upgrade (if you are not on it) to El Capitan. Your application should then build as normal. The only problem you will potentially face after building is the following: [INFO] : *** Terminating app due to uncaught exception 'InvalidOperationException', reason: 'fbauth2 is missing from your Info.plist under LSApplicationQueriesSchemes and is required for iOS 9.0' [INFO] : *** First throw call stack: This is unrelated to this ticket, but if appcelerator can answer how to resolve that, that would be great.
