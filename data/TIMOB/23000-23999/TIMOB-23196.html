Description

Problem

Debugging and setting expressions in the Expressions tab can cause the app to crash. Without the expressions it does not crash, but if you hit a breakpoint and have something in the expressions tab the app will crash in the simulator.

Test case

Create a new Mobile App Project in Studio using the Default Alloy Project template (Screenshot 2016-04-12 08.11.24.png and Screenshot 2016-04-12 08.12.44.png)

Set a breakpoint in the index.js file for the click event (Screenshot 2016-04-12 08.15.46.png)

Notice the breakpoint works and suspends the app (Screenshot 2016-04-12 08.16.23.png)

Enter in the Expressions tab the $.label.text that would be shown in the alert (Screenshot 2016-04-12 08.16.39.png)

App will crash in simulator (Screenshot 2016-04-12 08.16.39.png)
