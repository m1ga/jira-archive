  1. Hans Knöchel 2016-08-15 Ok, after some more research, I found the reason for the warnings. We reused certain proxies like Ti.UI.iPhone.TableViewCellSelectionStyle and Ti.UI.iPhone.ListViewCellSelectionStyle because they return the same values, so we had a class TiUIiPhoneTableViewCellSelectionStyleProxy that was used by both. Now we deprecated it, so only the warning for TableViewCellSelectionStyle was thrown. What we need to do is to create own classes to throw specific deprecated warnings, until we remove the classes in a later release (probably 7.0.0 or 8.0.0). I guess the same happens for SystemButton but will need to check that as well. PR (master): https://github.com/appcelerator/titanium_mobile/pull/8226 PR (5_5_X): https://github.com/appcelerator/titanium_mobile/pull/8227 Demo: 
       var win = Ti.UI.createWindow();
   
   var test1 = Ti.UI.iPhone.ListViewCellSelectionStyle.NONE;
