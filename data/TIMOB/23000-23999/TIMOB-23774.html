Description

- Team id is now required in the tiapp.xml when building an app with a watch extension and Xcode 8. - Added build configuration settings for swift files in watch apps Namely set the "SWIFT_VERSION" to "2.2", if it's not already set... all new Watch extension projects will have this set to "3.0" - Removed old watchos2 template in favor of existing watchos2-objc template. - Set default min iOS SDK version to 8.0 if not explicitly set in the tiapp.xml and compiling using Xcode 8. - Removed old maintainers from package.jsons.

  5. Chris Barber 2016-08-18 Team ids are returned by ti info, so Studio could provide a pretty dropdown in the tiapp.xml editor. In a future Titanium SDK release, we will most likely introduce a \-\-team\-id option instead of a \-\-pp\-uuid to start handling provisioning profiles the new Xcode 8 way. On the plus side, at least when you get the error, it does attempt to display a list of team ids. Unfortunately, the list of team ids comes from an a sqlite xcode cache database combined with the team ids parsed from all provisioning profiles. In other words, if you have a team, but you've never logged into Apple from Xcode's preferences and you don't have any valid, non-expired provisioning profiles installed, then we will report no teams found.
