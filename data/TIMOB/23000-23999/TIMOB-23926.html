Description

The Jenkins build job for Hyperloop sometimes chokes on a wrong Ti SDK. The CI build always trys to install the latest Ti SDK in conjunction with the -d flag to select it afterwards. If the latest Ti SDK is already installed this does nothing and we end up building with the Ti SDK that is currently selected on the build slave, which can be wrong.
