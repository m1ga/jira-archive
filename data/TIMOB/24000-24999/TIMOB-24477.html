I thought about this a while ago and a recent test case reminded me of it. We add the <enable-launch-screen-storyboard> by default to new projects, using the "LaunchScreen.storyboard" file. This is great, but it would be even greater if we would not need the tiapp flag for this anymore and instead be able to just check the existence of the file and decide whether to use it or not. That should be fine for the developer and also keeps the tiapp.xml more clean. [~cbarber] [~emerriman] thoughts?
