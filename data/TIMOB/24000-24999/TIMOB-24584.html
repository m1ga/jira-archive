  2. Joshua Quick 2017-10-06 Not a bug. You also need to set ScrollView property "contentWidth" to "Ti.UI.FILL", which sets the max horizontal size of the scrollable content area to the width of the ScrollView container. The default for "contentWidth" is AUTO, which causes the width of the content area to grow infinitely as you add views to the horizontal layout, and it'll never wrap because of this.
