Description

Let's be honest. Who is seriously using Ti.Yahoo API's these days? I guess some, probably even a lot. But not enough to reserve a seat in the SDK for it. That's why we should remove it from the core and provide the exact same classes / functionality in an external module. Roadmap:

Move to external module (ARC-based, move docs to module docs, link module docs to SDK docs

Deprecate it in 6.2.0, link the module to the core (like APSHTTPClient)

Remove from Core in 7.0.0

Objections in 3,2,1 ...
