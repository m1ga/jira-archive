Description

Description

Historically the winappdeploycmd exe used for Windows 10 device detection, app installation and such would always be installed 'globally' in C:\Program Files (x86)\Windows Kits\10}\bin\x86. Since 10.0.15063.0 this appears to be no longer the case and it is packaged with the SDK, at C:\Program Files (x86)\Windows Kits\10\bin\10.0.15063.0\x86 Our tooling makes the assumption that winappdeploycmd is in this global space, we need to adapt the tooling to inspect an SDK for winappdeploycmd, I'm unsure whether if we detect one we assign it to the overall VS info, overriding one that exists or we keep a winappdeploycmd tied to an SDK. Posted a question here to see whether this will be the norm from here https://docs.microsoft.com/en-us/windows/uwp/packaging/install-universal-windows-apps-with-the-winappdeploycmd-tool
