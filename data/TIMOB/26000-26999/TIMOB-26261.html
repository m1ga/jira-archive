Description

Update "titanium_mobile/android" to compile for Android P (ie: "compileSdkVersion" 28). This is needed in order to use the newest API Level 28 features so that we can work on the "Android P" epic. Starting off, Titanium should not target API Level 28 by default until all Android P breaking changes have been resolved. But we should allow the developer to opt-in to targeting API Level 28 to test the newest features/changes.
