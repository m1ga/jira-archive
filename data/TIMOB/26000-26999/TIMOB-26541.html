Description

According to the spec for setTimeout and setInterval for the web, the delay/interval value is optional and should default to 0 (or ASAP). https://developer.mozilla.org/en-US/docs/Web/API/WindowOrWorkerGlobalScope/setInterval The recent fixes on iOS around timers exposed the lack of support there, which was then fixed. Android also does not yet support omitting the delay argument. To hew to the spec as much as possible we'll create unspecified delays or delays of 0 or less as being 1ms delays.
