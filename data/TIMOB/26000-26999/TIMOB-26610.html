  13. David Jones 2020-02-03 We encountered the same issue for iOS when we added universal link support (eg https://ourdomain.com/xyz). The solution for us was to support universal links only and remove support for launching via app links (eg. ourdomain://launch-something/xyz). We would have preferred to be able to support both universal links and the deep link app links so that we could have more direct inter-app launching capabilities. We'll continue to watch this ticket.
