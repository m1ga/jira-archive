Comments

  1. Ewan Harris 2018-12-18 This is probably because of kitchensink-v2 having an app.xml file for internationalized string at the OS level, but not a strings.xml file which is perfectly valid. LiveView assumes that if you have an i18n directory, you've got a strings.xml file, we should just check for the existence of that file before attempting to read it.
  2. Ewan Harris 2018-12-18 PR: https://github.com/appcelerator/liveview/pull/121
  3. Ewan Harris 2019-01-08 To workaround this, create strings.xml file in the app/i18n/ de and en folders that dupes app.xml
  4. Samir Mohammed 2020-06-11 *Closing ticket* Fix verified in SDK version 9.1.0.v20200610180055.
