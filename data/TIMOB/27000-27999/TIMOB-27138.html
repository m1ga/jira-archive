Description

*Summary:* Android's Ti.UI.WebView does not currently support "capturing" a photo or video when using one of the below input HTML elements. It will display a photo gallery selection activity window instead. * * *To-Do:* When configuring our Intent in [TiWebChromeClient.java](https://github.com/appcelerator/titanium_mobile/blob/master/android/modules/ui/src/java/ti/modules/titanium/ui/widget/webview/TiWebChromeClient.java), we need to set it to [MediaStore.ACTION_IMAGE_CAPTURE](https://developer.android.com/reference/android/provider/MediaStore#ACTION_IMAGE_CAPTURE) for photo capture support and [MediaStore.ACTION_VIDEO_CAPTURE](https://developer.android.com/reference/android/provider/MediaStore#ACTION_VIDEO_CAPTURE) for video capture support. We also need to show a CAMERA permission request dialog as well. *Note:* We can only add support for Android 5.0 and higher.
