Description

Description

When building for iOS there is no module apiversion validation being performed, this is because the apiversion in the SDK is stored as iphone, and node-appc is looking for ios.

Steps to reproduce

1. Add the following module to your tiapp modules section com.appcelerator.aca 2. Open the manifest file (probably at /Users/eharris/Library/Application Support/Titanium/modules/iphone/com.appcelerator.aca/1.1.0/manifest) 3. Change the apiversion to anything other than 2 4. Build with ti build -p ios --build-only

Actual

Build carries on past the module verification stage

Expected

Build should error that module is incompatible
