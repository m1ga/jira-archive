Description

*Summary:* Adding a element within the "tiapp.xml" file's block will create an invalid entry within the generated "AndroidManifest.xml" when doing a build. *Note:* The entry is supposed to go within the block, not the block. So, doing this would have been invalid anyways. This makes this bug a very minor issue. https://developer.android.com/guide/topics/manifest/uses-library-element *Steps to reproduce:*

Setting a project's "tiapp.xml" with the entry below.

Build the project.

Open Finder or Window Explorer.

Go to project's subdirectory: ./build/android

Open the "AndroidManifest.xml" file.


http://ti.appcelerator.org">
	http://schemas.android.com/apk/res/android">
*Result:* The following invalid XML element can be found within the "AndroidManifest.xml" file. This doesn't prevent the app from running, but it's still not good that this is there. 
[object Object]
*Expected Result:* Build system should either omit the entry or add it as-is. *Solution:* As noted above, the is supposed to go within the block. So, all Titanium app devs should be doing the following instead. The build system will correctly inject this into the "AndroidManifest.xml" file. 

http://ti.appcelerator.org">
	http://schemas.android.com/apk/res/android">
