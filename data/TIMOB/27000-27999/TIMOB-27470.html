Description

Reproduce

1. Install nightly build 8.2.1.v20191014114554 or 8.2.1.v20191014062741 2. Create a brand new app, alloy or classic show this behaviour 3. Run the app with above SDK version

Problem

The app icon did not display, only the blank icon.

Note

One customer wants to use v8.2.1 including the deep link fix.

  4. Chris Barber 2019-10-17 The problem is related to TIMOB-27338 where we naively picked any iOS simulator that is compatible with the selected Xcode. However, if the first sim is an iPad (which it almost certainly is), then xcodebuild optimizes the asset catalog for iPad and thus iPhone app icons are stripped.
