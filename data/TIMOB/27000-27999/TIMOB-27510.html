Description

When evaluating JS passed in, the underlying code assumes the user JS code has no trailing semicolon and "blindly" appends one. If the user code *did* have a trailing semicolon, you would inadvertently get a null/undefined result rather than the value of the last expression/statement. See https://github.com/appcelerator/titanium-mobile-mocha-suite/blob/master/Resources/ti.ui.webview.test.js#L209-L211
