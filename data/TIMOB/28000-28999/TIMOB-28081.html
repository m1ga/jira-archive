Description

*Summary:* If an app includes an Apache "commons-logging" library via gradle (or a module that references it via gradle), then it will fail to build. *Steps to reproduce:*

Create a "Classic" Titanium app project.

Add the below "build.gradle" file to project folder: ./platform/android

Build for Android.

./platform/android/build.gradle 
dependencies {
	implementation 'commons-logging:commons-logging:1.1.3'
}

*Cause:* Titanium is including its own copy of the commons-logging-1.1.1.jar as shown below. https://github.com/appcelerator/titanium_mobile/tree/master/android/titanium/lib *Solution:* We should be referencing this library via gradle instead. This will allow gradle's dependency management resolve this for us. Or we can remove our usage of this JAR.
