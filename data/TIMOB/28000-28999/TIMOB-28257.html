Description

Description

When installing an SDK on Windows an EPERM error occurs when extracting the SDK, this is happening due to us now respecting symlinks on extraction. By default symlinks require administrator access to be created, or for [developer mode](https://docs.microsoft.com/en-us/windows/apps/get-started/enable-your-device-for-development) to be enabled. I think internally the SDK extraction code needs to check if we're on Windows and pass a type of junction to the symlink call To workaround this, run command prompt as an administrator and it will install just fine

Comments

  1. Ewan Harris 2020-11-25 Note to self, junction wont work as it's only for directories. We probably just want to copy the files
  2. Ewan Harris 2020-11-26 Can't copy the files directly as it's possible that we extract the symlink-ed file before we extract the source file. So we either need to just ignore symlinks completely on Windows, or defer the copy until we extract the source file later on
