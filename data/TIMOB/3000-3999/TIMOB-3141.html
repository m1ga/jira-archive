Description

Problem

A user can have multiple profiles that could be used to sign their app, even though they need a particular one to be used. Consider doing an adhoc build of an app using Urban Airship. The user has a generic team profile that does not include their push notification provisions, but they also have a specific profile that includes push notifications. Left alone, XCode is not choosing the best profile.

Workaround

The user edits their generated XCode project, changing the CODE_SIGN_IDENTITY to be what they want it to be.

Feature

Expose CODE_SIGN_IDENTITY through the tiapp.xml so that the user can specify which profile should be used.
