Comments

  1. Josh Roesslein 2012-08-31 A workaround would be to force each of these windows to be "heavyweight" so each window is its own activity. Right now if all of these are "lightweight" they share the same activity. Hitting back on android will close the top activity, closing all your LWs hosted by it. To force a heavyweight window you can create your window with one of the properties that triggers it. Fullscreen is one of those properties, so just set it: 
       createWindow({fullscreen: false}); // will create a heavyweight window
