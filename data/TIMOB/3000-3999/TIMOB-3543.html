Description

I've found that this error can occur if a project does not have a build/iphone directory. And the 'error' can be fixed by simply manually creating an empty build/phone.

Therefore, I suggest that Titanium Developer should either create this directory itself, or add the suggestion to create one, to the error message.

I got this error when checking a project out of source control, where the build directory is not checked in.
