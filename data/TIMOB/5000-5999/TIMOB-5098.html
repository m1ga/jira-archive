Description

Description: While testing Kitchensink, Base UI>Window (standalone), I opened the "Nav Covered" window and then rotated the device. The result was the new window was no longer covering the full width of the screen and showed the test window behind it. See screenshots. Occurs from landscape to portrait or vice-versa. Steps to reproduce: 1) Install KS 1.8.0 on iPod 3rd gen. 2) Navigate to Kitchensink: Base UI>Window (standalone) 3) Tap "Open (Nav Bar Covered)" button 4) Rotate device Result: The new window does not fully cover the test window Expected: The new window adjusts to the change in orientation Notes: Also occurs with SDK 1.7.2
