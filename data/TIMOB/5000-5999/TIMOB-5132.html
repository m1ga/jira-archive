Description

Description: While testing Kitchensink,Phone>Sound>Remote tests, I pressed the "stop" button and expected the stream to stop. When I started the stream again, the stream started again from the locations where I stopped it. This behavior is the same as the "pause" button. Steps to reproduce: 1) Install KS 1.8.0 on Nexus S 2) Navigate to Phone>Sound>Remote tests 3) Start the stream 4) Press "stop" 5) Start the stream again Result: The stream continues from where it was "stopped" Expected result: The stream resets and begins from the start of the stream again Notes: Also occurs with 1.7.2
