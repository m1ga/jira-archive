Description

Steps to reproduce: 1. Copy the sample code into a new ti mobile project 

var win = Ti.UI.createWindow({});
var xmlString = "";
var doc = Ti.XML.parseString(xmlString);
var elem = doc.getElementsByTagName("mickey").item(0);

win.open();
2. run the app Expected result: An error should be thrown because the element does not exist Actual result: No error is thrown

Comments

  1. Ivan Skugor 2011-12-08 I apologize for the interruption, but I'm not so sure that exception should be thrown in this case. If no element is found by some tag name, empty NodeList collection should be returned, so doc.getElementsByTagName("mickey").length would be 0 and doc.getElementsByTagName("mickey").item(0) should return null.
  2. Stephen Tramer 2011-12-12 This would violate the DOM L2 spec. Please see: http://www.w3.org/TR/DOM-Level-2-Core/core.html#ID-536297177 Node lists should return 'null', not throw an exception, for an item which is out of bounds.
  3. Lee Morris 2017-03-22 Closing ticket as "Won't Fix".
