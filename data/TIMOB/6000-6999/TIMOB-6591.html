Description

Description: In KS, on the Base UI>Views>Coverflow view>Coverflow Local Files test, scrolling to the far right side of the stack of images then switching tabs and returning results in rendering problems. In addition to the imediately visible rendering issue, there may be a an image partially rendered behind the center image that is visible as you scroll through the images. This image goes away as you scroll by the same image in the stack. Steps to reproduce: 1) Install KS built with SDK 1.8.0.1 2) Navigate to Base UI>Views>Coverflow view>Coverflow Local Files 3) Scroll to the far right of the stack 4) Switch tab 5) Switch back to the base UI tab Result: Image stack not as you left it with additional rendering problems as you manipulate the stack Expected: Images as you left them, no additional rendering issues Note: This also occurs on KS and SDK 1.7.5
