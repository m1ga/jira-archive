Description

If the Android Emulator is launched with any android version, then launched again using the same run configuration, but a different android version (one times out of three), the sd card fails to create and the new emulator does not launch. There is no studio log file. Console Log Attached. Steps to Reproduce: 1. Launch Studio, Create a New Project. 2. Delete Android emulator sd cards in .titanium folder. 3. Delete previous Android emulator run configurations. 4. Run the Android emulator. Expected Result: New emulator should launch. Actual Result: Console log with sd card argument errors. Emulator does not launch.
