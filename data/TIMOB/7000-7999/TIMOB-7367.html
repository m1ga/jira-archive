Description

Description: In KitchenSink iPad, while testing the "Main Tests -> Option Dialogs" I selected Dialog 1 and rotated the device with the option dialog displayed. While rotating, I noted two behaviors. The more common was a detaching of the Option dialog from the button that spawned it. Less common was a black non-functional options dialog. See screen shots. It was easiest to reproduce by rotating the device 270 degrees or more. Steps to reproduce: 1) Install KS iPad on iPad device 2) Launch KS iPad and choose "main tests" 3) Select options dialog 4) With the dialog up, rotate the device. Result: Frequent rendering of the options dialog in random screen locations, occasional black options dialog. Note: This also occurs with 1.8.0.1
