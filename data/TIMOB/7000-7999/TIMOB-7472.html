Description

It looks like we're including both the Rhino js.jar (via classes.dex) and the 2 V8 ABI-specific shared libraries (libkroll-v8.so, libstlport_shared.so) regardless of which runtime was chosen for the application in the tiapp.xml. We should only bundle the runtime that is given by the ti.android.runtime property in the tiapp.xml, or only bundle V8 by default. For V8 we should still be bundling both ABI-specific variations of the libraries in armeabi and armeabi-v7a folders. Q&A issue: http://developer.appcelerator.com/question/131313/why-are-both-rhino-and-v8-bundled-in-the-apk
