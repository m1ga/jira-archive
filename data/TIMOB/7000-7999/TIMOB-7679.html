Description

Description: When testing KitchenSink iPad, I clicked "Show popover" to enable a popover, and while it was displayed, rotated the device 270 degrees. The popover became disjointed from the button. I repeated the test with the remaining two buttons and found the same problem. In the case of "popover 1" the popover display also had some rendering problems resulting in its being unusable. See attached screenshots Steps to reproduce: 1) Install Kitchensink iPad on device built with SDK 1.8.2 2) Click "Open Main Tests" button 3) Click "Popovers" 4) Click "Show popover 1" 5) With the popover displayed, rotate the device 270 degrees Result: The popover may disappear. Continue to rotate the device in 90 degree increments and the popover should appear with rendering problems Expected: The popover remains usable and attached to the button that spawned it. Note: This also occurs in SDK 1.8.1 Repeat with other popovers for better illustration of the disjointed problem
