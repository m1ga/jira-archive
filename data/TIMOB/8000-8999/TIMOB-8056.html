Comments

  1. Allen Yeung 2012-03-21 This problem relates to linux having a case-sensitive file system. I'm working on a solution now, but you can workaround this issue by doing either of the following: 1. Create your module project with only lower-case characters. Or 2. In your test project, go into (modules > android > com.dmarie.tts > 2.0) and rename TTS.jar to tts.jar
  2. Allen Yeung 2012-03-22 A change was made to force module names to be lower-case during creation. To test the fix please try the following: 1. Create an android module with a name in caps (something like: titanium.py create --platform=android --type=module --name=TEST --id=com.test --android=/path/to/android/sdk 2. Go into the newly created module project, and build it by typing 'ant' 3. Go into the dist directory and verify that the name of the jar inside dist is all lower-case. (i.e. test.jar)
