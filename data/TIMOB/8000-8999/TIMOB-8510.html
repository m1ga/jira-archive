Description

The Accept-Encoding header is currently ignored when set on the HTTPClient. The following code has no affect on the header sent out - it always defaults to gzip xhr.open("POST", "http://url.tld/api_jsonrpc.php"); xhr.setRequestHeader("Accept-Encoding", ""); Looking at the generated code, gzip is always set to be the Accept-Encoding header.
