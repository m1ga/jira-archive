Description

When using KitchenSink-Nook from the beginning to Table Views > Append Row, then clicking on the Base UI tab, then clicking on Base UI > Views, the app screen turned black and touch events logged as 'No window to dispatch pointer action'. No force close dialog appeared. The app permanently had a black screen. App is force-closed when the back button is pressed. Unable to reproduce. Steps to Reproduce: 1. Run KitchenSink-Nook from Base UI > Tab Groups to Base UI > Views > Table Views > Append Row. 2. Go back to the main menu and press Base UI > Views. Expected Result: Views menu should load. Actual Result: Screen turns black and 'No window to dispatch pointer action' logs to console. App never actually hard crashes or force-closes.
