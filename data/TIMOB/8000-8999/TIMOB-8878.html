Steps to reproduce

Comments

  1. Hieu Pham 2012-04-25 This is an application bug. The app is trying to reuse tabs which are already closed. This would result in either a black screen (blank tab with no content) or a null pointer if Android collects the tab before the tabgroup's tabChanged callback is called. This is reproducible in 2.0.X branch as well as master.
