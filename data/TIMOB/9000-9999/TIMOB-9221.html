  1. Vishal Duggal 2012-07-31 Scrollable view has a chacheSize property which basically controls the number of views rendered and kept in cache. The lag/jitter is actually because of the backgroundImage loading and scaling (happens on the UI thread). At this point the device processing power comes into play. Ways to mitigate Use smaller resources for smaller form factor devices(The images in the project are all 640x960 which is obviously too large for anything other that an iPAD). Ways to fix Set the cacheSize to number of views which ensures all views are always rendered.
