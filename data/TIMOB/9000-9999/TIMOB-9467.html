Comments

  1. Chris Barber 2012-06-16 I tested with the latest master build of Mobile Web and was unable to reproduce the error on the Android 2.2 & 2.3 emulators and Android 2.3 device. Furthermore, the error will generally never seen by the end user. The error happens because we try to flush the analytics data when the page is unloaded. Some browser must have some sort of timeout. So, either we can choose to ignore this error and in most cases receive all analytics data when the page is unloaded -or- prevent the error by not sending analytics data on unload.
  2. Wilson Luu 2012-06-22 Reopening bug. Tested again on Droid 1 (2.2.3), sdk build 2.1.0.v20120621224153 and app is stuck on splash screen. This bug seems specific to Droid 1 (2.2.3)
