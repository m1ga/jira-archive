Description: During KitchenSink testing I noticed, that device's rotation causes changing size of a spinner. It happens when 2nd spinner scrolled down to an overlapped verb. This is not a regression, the behavior occurs on 2.0.2 as well. Steps to reproduce: 1. Run KitchenSink 2. Go to Controls - Picker - Android "useSpinner"-text2 3. Scroll down 2nd spinner with verbs until a word with overlapping 4. Rotate device Actual result: The spinner changes size Expected result: The spinner should not change size
