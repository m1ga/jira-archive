Description

Scrolling scrollable views doesn't work on WP7.5 because it doesn't support the events necessary to do dragging and it doesn't support animation. In this case, we should add buttons to the scrollable view that a user can tap to go forwards and backwards. We should consider making a new top-level API for managing these buttons.
