Description

List scroll for the sushi restaurants does not work after the first time we use it. Not a Regression, occurs with 2.0.2GA Steps to reproduce: 1.Build kitchen sink for mobile web in titanium studio. 2.Type the ip address of your laptop into the mobile device's web browser.(safari for iphone & IE for windows phone). 3.Hit go or enter. 4.Kitchen sink will fire up in the browser. 5.Tap on mashups tab. 6.Tap on the YQL table row. 7.Tap on the YQL local search. 8.You will see that you will get a list of sushi restaurants near appcelerator. 9.Now scroll the list. 10.Notice that you will be able to scroll the results for the first time,if you try to scroll the second time it does not scroll. Actual result: 1.List scroll for the sushi restaurants does not work after the first time we use it. Expected result: 1.Should be able to scroll the list of sushi restaurants smoothly even after the first time we scroll it.
